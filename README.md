# DLS_2_Project "LLM агенты"

### Части проекта:

Часть 1. Подготовка данных. Сохраняет данные в .csv файлы да последующей простой загрузки в датафреймы. 
В репозиторий я загрузил только один файл df_test_drop_01.csv, как пример. 
Все (df_data_drop_01.csv, df_train_drop_01.csv и df_val_drop_01.csv) подготовленные данные не смог сюда загрузить - сликом крупные.

Часть 2. Бэйзлайн. Предобученный трансформер дообучается на df_train_drop_01.csv, валидация после каждой эпохи на df_val_drop_01.csv. 
После завершения дообучения тестирование на df_test_drop_01.csv. 
Получаем точность бэйзлайна 73,20%. С этим значением будем сравнивать точность агента.

Часть 3. Агент с поиском. Два варианта: "..._with_zero_shot.ipynb" и "..._with_two_shot.ipynb".
Блокноты практически идентичные. Более подробные комментарии прописаны только во втором "..._with_two_shot.ipynb".

### Комментарии:

В файлах "..._with_zero_shot.log" и "..._with_two_shot.log" - логи работы соответствующих агентов.

В файле "search_results_cache.json" - кэш результатов поиска в Интернет (использовался для экономного расходования 1000 бесплатных поисков).

И логи, и кэш результатов поиска легко читаются глазами - можно анализировать.

### Чего удалось достичь:

Агент смог превзойти бэйзлайн, но незначительно.
Поиск смог улучшить результат агента без поиска, но незначительно. 
Лучшее качество агента с поиском: 76,20%. Выводы см. в блокноте "..._with_two_shot.ipynb".
